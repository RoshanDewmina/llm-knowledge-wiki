# When Should A Note Be Promoted?

## Why This Matters

- The vault should compound over time, which means temporary notes need a clear path into more durable project, synthesis, and output pages.

## Working Notes

- Promote a note when it captures a stable rule, a reusable explanation, a repeated answer, or a context summary that an agent should read again later.
- Leave a note in `wiki/journal/` when it is still exploratory, local to a single day, or missing enough evidence that formalizing it would create noise.

## Contradictions

- The desire to crystallize more knowledge can conflict with the need to keep the vault clean and non-duplicative.

## Promotion Target

- If the answer stabilizes, it should become part of a synthesis or project-memory rule rather than remain an isolated question note.
