# I made my OpenClaw agent more powerful by building my own harness

## Topic Map

- OpenClaw productivity depends on setup/investment.
- Four-lane dispatcher for direct/delegated/draft/block decisions.
- Agent mediums: built-in chat, Kanban/tasks, routines/cron.
- Dedicated agent computer/desktop, email, phone, card, connectors.
- Pluto architecture: SvelteKit, Convex, Daytona, Caddy gateway, Composio/Twilio/Supermemory/Mistral OCR/R2/observability.
- OpenClaw vs Hermes: author chose OpenClaw from familiarity but plans adapter for both.

## Useful Notes

- The central design is a triage dispatcher: quick tasks stay with the main agent; heavy tasks spawn subagents; outbound email/payment actions require human approval; abusive/nonsense tasks are blocked.
- Dedicated desktop is treated as observability and control: the user can see the agent's Linux desktop/browser/terminal and even take it over.
- Tasks are for large batches with progress states; routines are repeated scheduled jobs with notifications.
- Giving an agent email/phone/card makes it genuinely useful but also raises the need for explicit approval gates.
- Architecture stack is more product-harness than new agent core: the author deliberately builds around OpenClaw rather than building a new agent.

## Verified Claims

- The video presents a Pluto harness around OpenClaw with deployment, tools, desktop, email, phone, cards, and connectors.
- The author says human-in-the-loop is required for emails and payments.
- The author says he has not tested Hermes yet but is working on an adapter so users can choose OpenClaw or Hermes.
